# 3.6a Capability-Surface Teardown Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make a failed supervised unit tear down — terminate its OS process and mark its capability surfaces gone — and cascade that teardown to its direct dependents, while the shell and unrelated units keep running.

**Architecture:** A capability surface is modeled as a supervised out-of-process unit. Add a native-only `torn_down` flag to the supervisor's unit state. Teardown = terminate the process + set `kFailed + torn_down`, cascading one hop to direct dependents in reverse-dependency order. Teardown fires automatically when the restart budget drives a unit to `kFailed`, on intentional `StopUnit`, and via an explicit `TearDownUnit` API. Recoverable crashes still only degrade.

**Tech Stack:** C++20 (`host/supervisor.cpp`, `include/supervisor.hpp`), native contract tests compiled with `g++`/`clang++` and run from pytest (`tests/contracts/test_native_harness.py`).

## Global Constraints

- FROZEN — do not modify: `proto/capture.proto`, `include/plugin_system.hpp`, and the `plugins::RuntimeState` enum (no new enum value).
- `torn_down` is **native-only**: never surfaced over gRPC, never emitted as a `runtime_state` string, no `"UNLOADED"` value anywhere.
- Teardown cascade is **one hop**: the failed unit + its **direct** dependents only. No transitive multi-hop chain (that is a non-goal; deferred).
- Reverse-dependency order: direct dependents torn down **before** their provider.
- Auto-teardown fires **only** when the budget drives a unit to `kFailed`. A recoverable crash (`kRecovering`) leaves `torn_down = false` and only degrades dependents.
- `StopUnit` = intentional unload (teardown semantics), not degrade-only.
- `RestartUnit` success clears `torn_down` and returns `kRunning`; it does **not** restore dependents (that is task 3.7, out of scope).
- C++ and Python never share a process; supervised units are isolated OS processes.
- TDD: failing native test first, via the existing inline-`.cpp` + `g++` compile-and-run pattern. Tests short-circuit `return 0` under `#ifdef _WIN32`.
- Spec: `docs/superpowers/specs/2026-06-16-3.6a-capability-surface-teardown-design.md`.

```python
def _run_native_supervisor_test(
    tmp_path: Path,
    name: str,
    cpp_source: str,
    extra_sources: tuple[str, ...] = (),
) -> None:
    compiler = shutil.which('g++') or shutil.which('clang++')
    if compiler is None:
        pytest.skip('C++ compiler not available')

    source = tmp_path / f'{name}.cpp'
    binary = tmp_path / name
    source.write_text(cpp_source, encoding='utf-8')

    sources = [str(PROJECT_ROOT / 'host' / 'supervisor.cpp')]
    sources += [str(PROJECT_ROOT / s) for s in extra_sources]

    compile_result = subprocess.run(
        [
            compiler,
            '-std=c++20',
            '-I',
            str(PROJECT_ROOT / 'include'),
            *sources,
            str(source),
            '-o',
            str(binary),
        ],
        cwd=PROJECT_ROOT,
        check=False,
        capture_output=True,
        text=True,
    )
    assert compile_result.returncode == 0, (
        compile_result.stdout + compile_result.stderr
    )

    run_result = subprocess.run(
        [str(binary)],
        cwd=PROJECT_ROOT,
        check=False,
        capture_output=True,
        text=True,
    )
    assert run_result.returncode == 0, run_result.stdout + run_result.stderr
```

```cpp
    // Tear down a single unit: terminate its live process and mark it
    // unloaded (kFailed + torn_down). Idempotent.
    void TearDownOne(UnitRecord& record, std::chrono::milliseconds grace) {
        if (record.torn_down) {
            return;
        }
        if (record.process && record.process->IsAlive()) {
            record.process->Terminate(grace);
        }
        record.state = RuntimeState::kFailed;
        record.torn_down = true;
    }

    // Tear down a unit and its direct dependents (one hop), reverse-dependency
    // order: dependents quiesce before the provider.
    void TearDownCascade(UnitRecord& record, std::chrono::milliseconds grace) {
        if (record.torn_down) {
            return;
        }
        for (const UnitId dependent_id : record.direct_dependents) {
            TearDownOne(GetRecord(dependent_id), grace);
        }
        TearDownOne(record, grace);
    }
```

```python
        return 6;  // direct dependent now torn down
```
